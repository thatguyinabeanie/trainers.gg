Should opening a team have its own URL?

A

Route per team  ·  /builder/t/[id]  ·  recommended

Opening a team navigates to its own URL. Browser back returns to the list, links are shareable/bookmarkable, account teams load server-side. Local drafts get a stable local id (e.g. /builder/t/local-ab12). It's a real refactor — today's single /builder page splits into landing + editor routes.

B

In-place swap  ·  single /builder

The list and editor are one page; opening a team swaps the view in place with no URL change. Less to refactor, but no deep links, and the browser back button won't return you to the list (we'd fake it with an in-app "Back to teams").
