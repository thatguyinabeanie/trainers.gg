# Hero Page Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the root `/` redirect with a warm, community-focused landing page that advertises trainers.gg's features and shows live tournament data.

**Architecture:** Server Component page with client component islands for auth-aware CTAs. Tournament data fetched server-side with `unstable_cache` and streamed via Suspense. Static sections (hero, features, coming soon, closing CTA) render immediately.

**Tech Stack:** Next.js 16 App Router, Tailwind CSS 4, Lucide icons, existing shadcn/ui components (`Card`, `Button`, `StatusBadge`, `Skeleton`), Supabase queries via `@trainers/supabase`, `useAuthContext()` for client-side auth detection.

**Spec:** `docs/superpowers/specs/2026-03-24-hero-page-design.md`

### Task 1: Auth-Aware CTA Component

**Files:**

- Create: `apps/web/src/components/landing/hero-cta.tsx`

This is the client component that renders different CTAs based on auth state. Used in both the hero banner and closing CTA sections.

- [ ] **Step 1: Create the HeroCTA component**

```tsx
// apps/web/src/components/landing/hero-cta.tsx
"use client";

import { useAuthContext } from "@/components/auth/auth-provider";
import { buttonVariants } from "@/components/ui/button";
import { cn } from "@/lib/utils";
import Link from "next/link";

export function HeroCTA() {
  const { isAuthenticated, loading } = useAuthContext();

  if (loading) {
    // Render the unauthenticated CTAs while loading to avoid layout shift
    // (most visitors are unauthenticated anyway)
    return <CTAButtons authenticated={false} />;
  }

  return <CTAButtons authenticated={isAuthenticated} />;
}

function CTAButtons({ authenticated }: { authenticated: boolean }) {
  return (
    <div className="flex flex-col items-center gap-3 sm:flex-row sm:justify-center">
      {authenticated ? (
        <Link href="/dashboard" className={cn(buttonVariants({ size: "lg" }))}>
          Go to Dashboard
        </Link>
      ) : (
        <Link href="/sign-up" className={cn(buttonVariants({ size: "lg" }))}>
          Get Started
        </Link>
      )}
      <Link
        href="/tournaments"
        className={cn(buttonVariants({ variant: "outline", size: "lg" }))}
      >
        Browse Tournaments
      </Link>
    </div>
  );
}
```

**Pattern:** Uses `buttonVariants()` + `<Link>` (not `asChild` â€” Base UI does not support it). This matches the established pattern in `apps/web/src/app/invite/[token]/invite-error.tsx`.

- [ ] **Step 2: Create the tournaments preview component**

This component fetches upcoming tournaments server-side. Reuse the existing `listTournamentsGrouped` query and `UpcomingTournaments` display component from the tournaments page.

```tsx
// apps/web/src/components/landing/upcoming-tournaments-preview.tsx
import { unstable_cache } from "next/cache";
import { createStaticClient } from "@/lib/supabase/server";
import { listTournamentsGrouped } from "@trainers/supabase";
import { CacheTags } from "@/lib/cache";
import { UpcomingTournaments } from "@/components/tournaments/tournament-list";
import Link from "next/link";

export const revalidate = false;

const getCachedUpcoming = unstable_cache(
  async () => {
    const supabase = createStaticClient();
    const grouped = await listTournamentsGrouped(supabase, {
      completedLimit: 0,
    });
    return grouped.upcoming.slice(0, 5);
  },
  ["hero-upcoming-tournaments"],
  { tags: [CacheTags.TOURNAMENTS_LIST] }
);

export async function UpcomingTournamentsPreview() {
  const upcoming = await getCachedUpcoming();

  if (upcoming.length === 0) {
    return (
      <section className="py-16">
        <div className="mx-auto max-w-screen-xl px-4">
          <h2 className="mb-8 text-center text-2xl font-semibold">
            Upcoming Tournaments
          </h2>
          <p className="text-muted-foreground text-center">
            No upcoming tournaments right now. Check back soon!
          </p>
        </div>
      </section>
    );
  }

  return (
    <section className="py-16">
      <div className="mx-auto max-w-screen-xl px-4">
        <div className="mb-8 flex items-center justify-between">
          <h2 className="text-2xl font-semibold">Upcoming Tournaments</h2>
          <Link
            href="/tournaments"
            className="text-primary hover:text-primary/80 text-sm font-medium"
          >
            View All
          </Link>
        </div>
        {/* Reuses the existing UpcomingTournaments component which provides
            responsive table (desktop) / card list (mobile) with tournament name,
            org name, date, spots, and format info. No actionSlot needed â€”
            visitors can click through to register from the tournament page. */}
        <UpcomingTournaments tournaments={upcoming} showOrganization={true} />
      </div>
    </section>
  );
}
```

**Note on format display:** The spec mentions showing format. The existing `UpcomingTournaments` component already displays format labels in its table rows (via `TOURNAMENT_FORMAT_LABELS` in `tournament-list.tsx`), so this is covered by reusing the component.
