Tournament Management Page — Redesign

Currently 6 tabs nested inside the community dashboard. With the sidebar, the community nav moves out — but the tournament tabs remain. How should this page work?

Current issues

Triple navigation

Site nav + community tabs + tournament tabs. Three levels of horizontal navigation stacked vertically, eating ~200px before any content appears.

Heavy header

Tournament name + status badge + "Hosted by" + back arrow + "View Public Page" + community header above. Lots of chrome, little content.

Empty states dominate

Before Round 1 starts, the Overview, Pairings, and Standings tabs are mostly empty space with placeholder messages.

Settings is a mile long

Tournament settings is one massive scrolling form. During a live tournament, most of it is disabled anyway.

A

Keep horizontal tabs — simplify the chrome

The sidebar handles community nav, so we only need one tab bar for tournament tabs. Slim down the header. Same tab structure, less chrome.

  • Familiar — same 6 tabs, TOs already know where everything is
  • Sidebar eliminates one tab layer (community nav is gone)
  • Compact header with breadcrumbs — tournament name + status in one line

Weaknesses

B

Consolidated — 3 tabs + slide-out settings

Merge related concerns. Overview absorbs round control + stats. "Live" tab combines Pairings + Judge Queue + Standings (the things you need during a live tournament). Settings becomes a slide-out panel, not a tab. Audit Log moves to a button/dropdown.

Strengths

  • 3 tabs instead of 6 — focused on what TOs actually do
  • Overview shows everything at a glance — round control, judge queue, stats, top standings
  • "Live" tab = pairings + judge queue + full standings (the live ops view)
  • "Players" tab = registrations + drops (player management)

Weaknesses

  • More work to restructure — merging tab content
  • TOs familiar with current structure need to relearn
  • "Live" tab does a lot — needs careful sub-section design
💡 My recommendation: Option A for now, evolve toward B later. The sidebar already eliminates the triple-nav problem. Option A gives us a clean tournament management page with minimal restructuring — the 6 tabs work, they just need less chrome around them. The judge queue on the Overview tab is the key improvement. We can consolidate to 3 tabs in a future iteration once TOs have feedback on the sidebar layout.
