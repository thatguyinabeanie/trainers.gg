**Status:** Agent running (`feature-implementer`, opus, audit-first).

4. Write the revoke: `REVOKE SELECT … FROM anon, authenticated` on S-bucket base tables (excluding the realtime six and the public views which keep their anon/authenticated grant)

**Rationale for pause before running:** Every Wave A/B/C cached fetcher used `createStaticClient()` (anon key). The revoke as originally written would have silently broken those fetchers at runtime (invisible to typecheck). The fix order is swap-first, then revoke.
